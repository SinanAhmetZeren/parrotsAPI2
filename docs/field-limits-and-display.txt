Field Length Limits & Display Behavior
========================================

AGREED LIMITS (target)
-----------------------
Username          3–25
Waypoint title    25
Waypoint desc     300
Vehicle title     20
Voyage title      30
Voyage brief      300
Profile title     50
Bio               500


CURRENT LIMITS TABLE
---------------------
Field               Backend     Mobile App     Web
-----------------------------------------------------------
Username            3–25 ✓      25 ✓           25 ✓
Waypoint title      25 ✓        25 ✓           25 ✓
Waypoint desc       300 ✓       300 ✓          300 ✓
Vehicle title       20 ✓        20 ✓           20 ✓
Voyage title        30 ✓        30 ✓           30 ✓
Voyage brief        300 ✓       300 ✓          300 ✓
Profile title       50 ✓        50 ✓           50 ✓
Bio                 500 ✓       500 ✓          500 ✓

Notes:
- Only username is consistently validated across all three platforms.
- Mobile has limits on all fields but most differ from agreed targets above.
- Web has no maxLength on any field except username (register screen).
- Backend has no validation except username. DB uses type: "text" (unlimited) for all other fields.
- Need to apply agreed limits to: backend DTOs/controllers, web inputs, and update mobile to match.


DISPLAY & TRUNCATION TABLE
---------------------------
Field                   Mobile                              Web
--------------------------------------------------------------------------------------------
Voyage title            2 lines, no read more               Full (with tooltip)
Voyage brief            3 lines, no read more               4 lines, no read more
Voyage desc (detail)    —                                   Read more / Read less ✓
Vehicle title           Hard cut 16 chars + "..."           Hard cut 18 chars + "..."
Vehicle description     4 lines, no read more               5 lines, no read more
Waypoint title          Full                                Full
Waypoint description    Full (no truncation)                7 lines, no read more
Profile title           Hard cut 35 chars + "..."           Full
Bio                     Read more / Read less ✓ (200 chars) Scrollable, full display
Username                Hard cut 30 chars + "..."           Full

Issues & Notes:
- Vehicle title hard-cut with substring is fragile — long names get chopped mid-word.
  Better to use numberOfLines={1} + ellipsizeMode="tail" (mobile) or CSS text-overflow (web).
- Username hard-cut at 30 on mobile is pointless — max is 25 so it never triggers.
- Bio on web should have read more like mobile, especially once 500 char limit is applied.
- Waypoint description on mobile shows full with no truncation — could get very long on screen.
- Voyage description detail page on web has read more, but mobile has no equivalent (goes to detail screen instead).
