3 ERRORS TO FIX
===============

1. PROFILE SCREEN REFETCH ON LOGOUT
   Error:   "Cannot refetch a query that has not been started yet."
   Cause:   useFocusEffect fires during logout navigation transition when userId
            is already null, causing RTK Query to attempt refetch on uninitialized queries.
   Fix:     Add userId guard to refetch calls in ProfileScreen.jsx:
            if (!userUninit && userId) await refetchUserData();
            if (!voyageUninit && userId) await refetchVoyageData();
            if (!vehicleUninit && userId) await refetchVehicleData();

2. SESSION EXPIRED TOAST ON INITIAL LAUNCH / LOGIN SCREEN
   Error:   "Session expired - Please log in again." toast shown on fresh app launch
            or on the login screen.
   Cause:   TokenExpiryGuard is rendered in LoginScreen.jsx. On focus it checks
            AsyncStorage for the refresh token. On fresh install there is no token,
            so !expiry is true and it incorrectly shows the toast and dispatches logout.
   Fix:     1. Remove TokenExpiryGuard from LoginScreen.jsx entirely — it has no
               business being on a screen where the user is not logged in.
            2. In TokenExpiryGuard.jsx, add isLoggedIn guard so the toast and logout
               dispatch only fire when the user is actually logged in and the token
               has genuinely expired.

3. "NO CONNECTION - RETRYING..." PILL SHOWN AFTER LOGIN
   Error:   The retrying banner appears too early after login or after a brief
            hub disconnection blip.
   Cause:   App.js shows the pill after only 4 seconds of isLoggedIn && !isHubConnected.
            This is too short — it fires on initial hub connection (hub needs time to
            establish after login) and on momentary reconnection blips.
   Fix:     Increase the delay from 4000ms to 20000ms so the pill only appears when
            the hub has been consistently disconnected for 20+ seconds while logged in.
            Also reset isHubConnected to false in updateAsLoggedOut so the timer
            clears cleanly on logout and does not carry over stale state.
