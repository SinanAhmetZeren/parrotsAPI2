GROUP UNREAD BADGE FIX
======================

BUG
---
When a user is actively viewing a group conversation and another member sends a message,
the unread badge incorrectly appears. The backend fires ReceiveUnreadNotification without
checking whether the recipient is already viewing that group.

ROOT CAUSE
----------
SendGroupMessage only checks isOnMessagesScreen before setting _unreadCache and sending
ReceiveUnreadNotification. There is no isViewingThisGroup check, unlike the DM flow which
checks isViewingConversation.

PATTERN (how DMs work correctly)
---------------------------------
- EnterConversationPage  → tracks screen only, NO cache change
- EnterMessagesScreen    → clears _unreadCache[userId] = false + clears DB
- SendMessage            → sets cache + sends notification only when !isViewingConversation && !isOnMessagesScreen

FIX
---

Backend - Chathub.cs:
  1. Add EnterGroupConversationPage(string userId, string groupId)
       → _tracker.EnterConversation(userId, Context.ConnectionId, groupId.ToString())
       → No cache change (matches DM enter behavior)

  2. Add LeaveGroupConversationPage(string userId)
       → _tracker.LeaveConversation(Context.ConnectionId)
       → Reuses existing method

  3. OnDisconnectedAsync already calls LeaveConversation(connectionId) — no change needed

  4. In SendGroupMessage, change:
       bool shouldNotifyUnread = memberId != senderId && !isOnMessagesScreen;
     To:
       bool isViewingThisGroup = _tracker.IsViewingConversation(memberId, groupConversationId.ToString());
       bool shouldNotifyUnread = memberId != senderId && !isOnMessagesScreen && !isViewingThisGroup;

Backend - ConversationPageTracker.cs:
  - No changes needed
  - Reuse _activeConversation dictionary, storing groupId in the PartnerId slot
  - GroupIds are ints (e.g. "42"), UserIds are GUIDs — no collision risk

Frontend - GroupConversationDetail.jsx:
  - On focus: invokeHub("EnterGroupConversationPage", currentUserId, groupId)
  - On blur:  invokeHub("LeaveGroupConversationPage", currentUserId)
